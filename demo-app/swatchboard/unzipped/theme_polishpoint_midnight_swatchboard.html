Midnight Swatch Board

Every component in the PolishPoint Midnight theme, rendered with real app class names and real CSS. Components are organized into size tiers — shared visual specifications (Container, Block, Row, Tile) that multiple functional classes reuse. When adding new components, pick the tier that matches the size you need; don't invent new dimensions.

Toggle the variant switcher to see how icolor-*, inner-*, shadow-*, and cards-* body classes compose on variant-aware tiers. Duplicate this file, swap the :root tokens and icolor-* blocks to build a new theme.

3. Container — Tier

Tier definition
Standalone outer shell with a title. Holds Blocks, Rows, or arbitrary content. This is your "outer card" — the titled section wrapper that defines page regions like "Today's Schedule" or "Field Ops".
Padding28px (all sides)
Border radius24px
Backgroundvar(--card-bg) — dark #1E1D1B, flat (not a gradient)
Border1px solid var(--card-border) #2E2C28, border-top: none (gold hairline replaces top edge)
Defining accentGold hairline via ::before pseudo-element (inset 8% on each side)
Box shadowNeumorphic stack: 8px 8px 22px rgba(0,0,0,0.45), -3px -3px 12px rgba(255,255,255,0.03)
Where it livesDirectly in the page layout (columns, grids, full-width). NOT nested inside anything.
Has a title?Yes — via <div class="dash-card-title"> as the first child
Variant participationcards-* (edge treatments), shadow-* (depth). NOT affected by icolor-* or inner-*.

5. Row — Tier

Tier definition
Nested inside a Container. Pill-shaped list row with an alternating zebra pattern (solid var(--primary-light) bright gold on odd rows, subtle white 6% on even rows) — gives the list a premium "chip" feel where each row reads as a discrete item. Odd rows invert their text color to dark charcoal #2A1F08 with a warm-white text-shadow for readability on the bright gold fill. Uses the :nth-child(odd of .check-item) CSS4 selector so row counting ignores preceding siblings like .dash-card-title. Added 2026-04-10, brightened and converted to pill form on the same day. Used for notifications, client lists, team rosters, anywhere you want a flat list inside a Container.
Padding10px vertical, 18px horizontal
Border radius12px — rounded rectangle matching Block tier (matches .inner-lifted look shape, less aggressive than pill)
Background (odd rows, incl. top)var(--primary) — warm gold #C9A84C
Background (even rows)var(--primary-light) — lighter warm gold #D4B96A
Text color (odd rows)#2A1F08 dark charcoal with text-shadow: 0 1px 0 rgba(255,255,255,0.22) for readability on the bright gold fill. Cascades to strong, .text-primary, .text-body, .text-muted, .text-sm, .text-xs.
BorderNone — shape is now delimited by the pill, no dividers needed
Box shadowNone
Layoutdisplay: flex, align-items: center, gap: 12px (horizontal)
Margin6px 0 between pills for vertical breathing room; :first-child and :last-child zeroed out so the stack ends cleanly inside its Container
Row counting:nth-child(odd of .check-item) — CSS Selectors Level 4 syntax that counts only among .check-item siblings, ignoring preceding titles or wrapper children. Supported in Chrome 111+, Firefox 113+, Safari 9+, Edge 111+.
Font size13px
Where it livesInside a Container (stacked in lists)
Has a title?No — one line item per row
Variant participationNONE — stays neutral regardless of body classes. Intentionally excluded from the icolor-* / inner-* / shadow-* / cards-* variant system. The zebra + pill treatment is a fixed Row-tier signature, not a variant.
